Furuta Pendulum

A rotary inverted pendulum built to turn control theory into something physical. I used an Arduino Uno, stepper motor, encoder feedback, and LQR control to balance the pendulum upright.

Control Systems Arduino Uno ATmega328P LQR Embedded C Debugging
Why I built it

After finishing COMPSYS 201, I wanted to stop only memorising theory and actually use it. That course taught me the basics of the ATmega328P, which is the same microcontroller used in the Arduino Uno R3. This made the Arduino a good choice because I could connect what I had learned about registers, interrupts, timing, and embedded systems to a real project.

The Furuta pendulum felt like the right challenge: difficult enough to need real control theory, but still feasible to build with parts I could access.

What I did

I built a rotary inverted pendulum using a NEMA17 stepper motor, a TMC2209 driver, a quadrature encoder, and an Arduino Uno. The encoder measures the pendulum angle while the stepper motor rotates the arm to keep the pendulum balanced upright.

I first tried an inner and outer PID controller, but it was difficult to tune and did not give the stability I wanted. I then switched to an LQR controller, which worked much better because it controls the system using the full state instead of reacting to only one error at a time.

Debugging

One of the hardest issues was noisy encoder readings. I used an oscilloscope to check the encoder signals and narrow down whether the issue was wiring, pull-ups, power, or code.

The fault ended up being a bad USB cable connected to the Arduino. Replacing it fixed the signal problem and made the controller behave much more consistently.

What I learned

This project helped me connect embedded programming, motor control, signal debugging, and control theory in one build. I learned that a controller is only as good as the hardware signals feeding it, and that debugging the physical system matters just as much as the code.

Next step

The pendulum currently balances once it is placed upright. My next goal is swing-up control, where the system brings itself from the hanging position to upright automatically.

I want to explore reinforcement learning for this using MuJoCo, MATLAB, and Simulink. The plan is to train and test the swing-up behaviour in simulation first, then combine it with the existing LQR controller for upright balancing.
